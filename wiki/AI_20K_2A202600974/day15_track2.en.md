# Track 2: AI Infrastructure & Data (Phase 2)

## Overview

Track 2 trains infrastructure and AI platform engineers, focusing on the ability to build and operate the backend of an AI system at production level.

Main content includes:

- Lakehouse & Feature Store.
- vLLM deployment & optimization.
- CI/CD for AI (LLMOps).
- *CI/CD cho AI (LLMOps).*
- GPU FinOps & cost management (especially token cost).
- Production data pipeline.

## Relevant Signals

This track is suitable if you:

- In Phase 1 (Day 7, Day 10, Day 12), you were interested in building data pipelines, observability, and deployment.
- You are familiar with and enjoy working with CLI / Docker / CI.
- You have a mindset for SLA / SLO / P95 / Cost.
- You prefer building stable and reliable systems rather than adding new features.

## Capabilities to Build

- Cloud & Infrastructure as Code (IaC).
- *Cloud & Infrastructure as Code (IaC).*
- Lakehouse, Vector Store, Feature Store.
- *Lakehouse, Vector Store, Feature Store.*
- Serving & Scaling (deploying large models).
- CI/CD for AI environments.
- Observability / LLMOps.
- *Observability / LLMOps.*
- Governance, GPU management & FinOps.
- *Governance, GPU management & FinOps.*

## Difficulties / Challenges

- Infrastructure knowledge is very broad, requiring patience with operational details.
- Few "pretty" things or UI/UX to demo.
- Pressure regarding system reliability (on-call).
- DevOps learning curve is steep, requires a solid foundation.
- **Hard requirements:** Need GPU or Cloud free-tier to practice labs on serving and FinOps.

## Career Path & Opportunities

- **Target roles (CP2):** AI Data Engineer, Platform Engineer, MLOps Engineer.
- **Market demand:** **Very high.** Especially as enterprises want to own models, optimize costs, and bring AI into production instead of just using APIs.
