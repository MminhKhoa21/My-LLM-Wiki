# Day 24 Track 3: RAGAS, LLM-as-Judge & Guardrails  

This page summarizes the lecture and lab on RAG Evaluation and Guardrail Systems for Track 3.  

## Evaluation Foundations  

- **80/5 Rule:** Teams historically spend 80% of their time building and 5% evaluating. The 2026 standard shifts this to 50% building, 30% evaluating, and 20% guardrails. Manual vibe-checks do not scale.  
- **Offline vs. Online Eval:** Offline evals act as CI gates before deployment using static datasets. Online evals sample production traffic for drift detection and monitoring.  
- **Component vs. End-to-End:** RAGAS provides component-level evaluation (retrieval vs. generation), while LLM-as-Judge provides end-to-end holistic scoring.  

### RAGAS: 4 Core Metrics  

1. **Faithfulness:** Does the context support the answer? Checks for intrinsic hallucinations. (Target $\ge 0.85$)  
2. **Answer Relevancy:** Does the answer address the question? Uses reverse question generation to penalize off-topic answers. (Target $\ge 0.80$)  
3. **Context Precision:** Are the relevant retrieved chunks ranked at the top? Evaluated using NDCG. (Target $\ge 0.70$)  
4. **Context Recall:** Does the retrieved context cover the ground truth? Requires a reference answer. (Target $\ge 0.75$)  

### LLM-as-Judge  
### LLM-as-Judge  

Used to scale evaluations from hundreds to hundreds of thousands of queries, but requires mitigation of four main biases:  

1. **Position Bias:** Models favor the first or last option. Mitigated via *swap-and-average*.  
2. **Length Bias:** Models favor verbose answers. Mitigated by length-controlled evals or strict rubrics.  
3. **Self-Enhancement Bias:** Models prefer outputs generated by themselves. Mitigated via *cross-judge protocols*.  
4. **Style Bias:** Models prefer formatted outputs (e.g., markdown) over plain text. Mitigated by stripping formatting before judging.  
- **Human Calibration:** It's essential to measure the judge's alignment with human raters using **Cohen's Kappa**. A score $\ge 0.60$ is required for production.  

### Hallucination Detection  

- **SelfCheckGPT:** Samples multiple answers with temperature > 0 and measures consistency. High variance indicates hallucination.  
- **NLI (Natural Language Inference):** Uses models like DeBERTa to verify if the retrieved context entails the answer.  
- **Semantic Entropy:** Clusters answers by semantic equivalence to measure uncertainty and hallucination probability.  

## Guardrails  
## Guardrails  

Guardrails provide a defense-in-depth architecture across four dimensions: Topical, Safety, Security, and Compliance.  

- **L1 Input Layer (<30ms):** Validates input using PII redaction (Presidio), Prompt injection detection (Prompt Guard), and Topic scope validators (Guardrails AI).  
- **L2 LLM Layer (0ms):** Handled via structural prompts.  
- **L3 Output Layer (<50ms):** Classifies safety (Llama Guard 3) and detects hallucinations (NLI).  
- **L4 Audit Layer:** Async logging of all queries and actions.  

### Attack Patterns  

- **Direct Injection:** e.g., DAN (Do Anything Now), Payload splitting, Encoding bypass.  
- **Indirect Injection:** Malicious payloads planted in retrieved documents.  
- **Session Poisoning:** Attacker injects malicious intent into the conversation history, then uses a benign query to trigger the exploit, bypassing simple output blockers. Defended by replacing malicious inputs in the history itself.  

## Lab 24: Full Evaluation & Guardrail System  

The lab has four phases:  

- **Phase A (RAGAS):** Generate synthetic test sets, run the 4 RAGAS metrics, analyze failure clusters, and set up a CI/CD GitHub Action gate.  
- **Phase B (LLM-as-Judge):** Build pairwise and absolute scoring pipelines, measure Cohen's Kappa against human labels, and analyze judge bias.  
- **Phase C (Guardrails):** Implement a full stack guardrail system with Presidio, topic validators, adversarial testing, and Llama Guard 3, benchmarked for P50/P95 latency.  
- **Phase D (Blueprint):** Document the system architecture, Service Level Objectives (SLOs), alert playbooks, and cost analysis.
