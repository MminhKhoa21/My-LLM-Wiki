LangGraph HITL — Code-Level
from langchain.agents import create_agent
from langchain.agents.middleware import
HumanInTheLoopMiddleware
from langgraph.checkpoint.memory import InMemorySaver
from langgraph.types import Command
agent = create_agent(
model="gpt-5.5",
tools=[write_file, send_email, ask_user],
middleware=[HumanInTheLoopMiddleware(
interrupt_on={
"write_file": { "allowed_decisions": [ "approve"
, "edit", "reject"]},
"send_email": { "allowed_decisions": [ "approve"
, "edit", "reject"]},
"ask_user": { "allowed_decisions": [ "respond"
]},
})],
checkpointer=InMemorySaver(),
)
agent.invoke(Command(resume={"decisions": [{
"type": "edit",
"edited_action": { "name": "send_email", "args": { "to":
"ops@vinuni.edu.vn"}}
}]}), config=config)
3 cập nhật quan trọng:
1. Policy per tool: tool nào cho
approve/edit/reject/respond
2. Checkpointer bắt buộc: pause
phải survive refresh / reconnect
3. Respond dành cho “ask human”
tool, không dùng để từ chối
action nguy hiểm
Lưu ý: Nếu edit args quá mạnh,
model có thể re-plan lại. Vì vậy
approval UI nên khuyến khích
small edits thay vì viết lại toàn
bộ action.

Audit Trail — Mỗi quyết định đều được ghi lại
class AuditEntry(BaseModel):
timestamp: datetime
agent_id: str
action: str
confidence: float
risk_level: str # low/med/high
reviewer_id: str | None
decision: str # auto/approve/reject
reason: str | None
execution_time_ms: int
# Immutable: append-only PostgreSQL
# Backup: S3 daily snapshot
# Replay: reconstruct full session
Mỗi event ghi:
■ Who: agent nào, reviewer nào
■ What: action gì, confidence bao
nhiêu
■ When: timestamp chính xác
■ Why: reasoning của agent +
decision
Lưu ý: Audit trail là compliance
requirement (GDPR, SOC2).
Build from day one — cannot
retrofit sau khi đã production.

Decision Analytics — Đo lường để cải thiện
Approval rate
bao nhiêu action được duyệt?
Review latency
mất bao lâu để trả lời?
Regret / override
đã duyệt rồi có phải rollback?
Ít regret + latency thấp
→nới autonomy
3 metrics mới nên theo dõi:
1. Approval rate: quá thấp → agent
đang xin phép sai chỗ
2. Review latency: cao → form/UI
chưa đủ rõ hoặc hỏi quá nhiều
3. Regret rate: human approve xong
vẫn rollback → gating chưa đủ tốt
Chỉ tăng autonomy khi approval
cao và regret thấp.
Nếu approval cao nhưng regret
cũng cao, nghĩa là reviewer đang
approve “cho xong”.
Đo trust bằng hành vi, không đo
bằng cảm giác.
