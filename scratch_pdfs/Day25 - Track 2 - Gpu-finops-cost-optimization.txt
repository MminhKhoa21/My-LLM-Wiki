GPU Cloud Cost Anatomy
Hiểu bill GPU không chỉ là giá card theo giờ; hidden cost,
idle cost, và over-provisioning mới là nơi thất thoát ngân
sách lớn nhất.

Chi Phí GPU Cloud: Breakdown
Compute (GPU hours) 60% Storage 15%Net 10% Other 15%
■ Hidden costs: data transfer egress ($0.09/GB AWS), NAT gateway ($0.045/GB), Secrets Manager
($0.40/secret/mo)
■ Wasted spend: idle GPUs (training done, instance running), over-provisioned instances, unused reserved
capacity
■ FinOps maturity: Inform → Optimize → Operate — hầu hết AI teams chỉ ở Inform level

Spot Instances: Tiết Kiệm 60–70%
■ Discount 60–70% so với on-demand
■ 2-min termination notice
■ Spot Fleet: request từ nhiều AZ/instance
types
■ Giảm interruption rate từ 15% → 3%
■ Discount tới 80%
■ Terminate sau 24h (Preemptible) hoặc
flexible (Spot)
■ Phù hợp jobs <20h
■ T ự động reschedule trên GKE
Mixed fleet strategy: 20% on-demand (baseline) + 80% spot (burst) — balance cost vs reliability SkyPilot:

chọn provider tối ưu.

Cost/Giờ vs Cost/T oken: Chốt KPI Nào?
■ Dễ hiểu cho budgeting và chargeback
■ Hợp để soi idle time, spot mix,
right-sizing
■ Nhưng chưa phản ánh tốc độ sinh token
■ Hai cụm cùng $10/h có thể khác xa hiệu
quả

3600)

engine

khi cost/giờ giữ nguyên
■ Phù hợp hơn cho bài toán tối ưu inference
LLM

tiêu.

■ Prefill xử lý prompt dài → thiên về

■ Decode sinh token tuần tự → thiên về
memory bandwidth
■ Dồn cả hai vào cùng GPU thường làm
nghẽn tài nguyên
■ Prompt càng dài, lệch pha càng rõ

■ Hợp với long-context, agentic, RAG,
reasoning
■ Hỗ trợ stack như TensorRT-LLM, vLLM,
SGLang

Chunked Prefilling: Cân Bằng TTFT và ITL
■ Chia prompt dài thành nhiều chunk nhỏ
■ Ưu tiên decode requests trước
■ Phần token budget còn lại mới nạp thêm
prefill
■ Tránh để prompt dài chặn hàng đợi
decode
■ Tốt cho traffic hỗn hợp: prompt dài +
chat liên tục
■ Giảm tail latency và tăng độ mượt khi
sinh token
■ Chunk quá nhỏ → TTFT xấu đi
■ Chunk quá lớn → decode vẫn bị chen
ngang
Rule of thumb: bắt đầu tune theo max_num_batched_tokens, đo đồng thời TTFT, ITL, queue depth.

Speculative Decoding: Giảm Độ Trễ Sinh T oken
■ Một draft model dự đoán trước vài token
■ Model chính xác minh cả cụm token trong
một lượt
■ Nếu khớp nhiều → ITL giảm rõ rệt
■ Hợp với workload decode đang
memory-bound
■ Hiệu quả cao hơn ở medium/low QPS,
latency-sensitive
■ Cần thêm draft model hoặc MTP/EAGLE
backend
■ Không phải lúc nào cũng tăng throughput
giờ cao điểm
■ Theo dõi acceptance rate và tok/s thực
tế
Đừng bật mù quáng: nếu acceptance thấp, bạn chỉ đang trả thêm chi phí xác minh mà không thu được tốc độ.

KV Cache & Prefix Caching
■ Lưu key/value của các token đã xử lý
■ Decode token tiếp theo không phải tính
lại toàn prompt
■ Đổi lại: ăn VRAM, dễ tạo áp lực
admission/eviction
■ Có thể offload sang host nếu engine hỗ
trợ
■ Tái sử dụng phần đầu prompt giống nhau
giữa nhiều request
■ Rất hợp với system prompt dài, tài liệu
lặp lại, RAG cố định
■ Với NVIDIA NIM, bật
NIM_ENABLE_KV_CACHE_REUSE=1
■ Cache hit tốt → TTFT request thứ 2 giảm
mạnh
Metric bắt buộc: cache hit rate, KV memory usage, eviction rate, TTFT trước/sau khi bật cache.
